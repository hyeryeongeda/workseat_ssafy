**자바스크립트 DOM 수업 정리 노트**

1. **자바스크립트 역사와 웹 브라우저**

   * 웹은 ECMAScript 표준과 브라우저 경쟁을 통해 발전.
   * IE → Firefox → Chrome 순으로 발전하면서 현재의 빠르고 동적인 웹 환경이 구축됨.

2. **ECMAScript와 자바스크립트**

   * **ECMAScript**: 자바스크립트의 표준 규격을 정의하는 규칙서.
   * **자바스크립트**: ECMAScript 규격을 기반으로 구현된 언어.
   * ES6 이후, 자바스크립트는 웹을 넘어 다양한 분야에서 핵심 언어로 확장됨.

3. **변수 작성 규칙**

   * **기본 규칙**: 변수는 문자, 달러( $ ), 밑줄( _ )로 시작. 대소문자 구분되며, 숫자나 공백, 특수문자 포함 불가.

   * **예약어 사용 금지**: 예시로 `for`, `if`, `function`, `class`, `return`, `let`, `const` 등이 있음.

   * **좋은 변수명 작성법**: 의미가 드러나는 이름을 사용 (e.g., `userName`).

   * `$` 또는 `_`로 시작하는 변수는 특별한 용도로 사용, 일반적으로는 피하는 것이 좋음.

4. **네이밍 규칙**

   * **카멜 케이스 (camelCase)**: 변수명, 함수명에 가장 많이 사용 (e.g., `userName`).
   * **파스칼 케이스 (PascalCase)**: 클래스나 생성자 함수 이름에 사용 (e.g., `UserService`).
   * **스네이크 케이스 (SNAKE_CASE)**: 상수에 사용 (e.g., `API_KEY`).

5. **변수 선언 키워드**

   * **`let`**: 재할당 가능, 재선언 불가, 블록 스코프.
   * **`const`**: 재할당 불가, 재선언 불가, 선언 시 초기화 필수, 블록 스코프.
   * **`var`**: 재할당 가능, 재선언 가능, 함수 스코프. 호이스팅 문제로 사용 비권장.

6. **`let`과 `const`의 특징**

   * **`let`**: 값을 재할당할 수 있지만 재선언은 불가능 (e.g., `let number = 10;`).
   * **`const`**: 값의 재할당이 불가능하며, 선언 시 반드시 초기화해야 함 (e.g., `const number = 10;`).

7. **블록 스코프 (Block Scope)**

   * **블록 스코프**: `let`과 `const`는 블록 내에서만 유효. `var`는 함수 스코프.
   * **`let`/`const`**: 예측 가능한 코드로, 블록 외부에서 접근 불가.
   * **`var`**: 블록 무시하고 함수 범위에서만 유효.

8. **어떤 키워드를 사용해야 할까?**

   * **기본 원칙**: `const`를 기본으로 사용.
   * **필요 시 `let` 사용**: 반복문 등 값 변경이 자연스러운 경우에만 사용.

9. **DOM (Document Object Model)**

   * **Document 객체**: HTML 문서의 구조를 자바스크립트로 제어할 수 있게 해줌.
   * **DOM 트리**: HTML 문서의 구조를 트리 형태로 표현.
   * **DOM 선택**: `getElementById`, `getElementsByClassName`, `querySelector` 등으로 HTML 요소를 선택.
   * **DOM 조작**: 선택한 요소의 속성, 콘텐츠, 스타일 등을 자바스크립트로 변경 가능.

**주요 키포인트**:

* 자바스크립트의 변수 선언은 `let`, `const`를 사용하는 것이 좋으며, `const`는 값이 변경되지 않는 변수에 사용합니다.
* DOM 조작을 통해 HTML 문서를 동적으로 변경할 수 있습니다.
